### JARVIS OS — Distributed AI Operating System

[![Repo](https://img.shields.io/badge/GitHub-jarvis--linux-181717?style=flat-square&logo=github)](https://github.com/Turbo31150/jarvis-linux)
[![Language](https://img.shields.io/badge/TypeScript-3178C6?style=flat-square&logo=typescript&logoColor=white)](https://github.com/Turbo31150/jarvis-linux)
[![Python](https://img.shields.io/badge/Python-3776AB?style=flat-square&logo=python&logoColor=white)](https://github.com/Turbo31150/jarvis-linux)
[![CUDA](https://img.shields.io/badge/CUDA-76B900?style=flat-square&logo=nvidia&logoColor=white)](https://github.com/Turbo31150/jarvis-linux)
[![Stars](https://img.shields.io/github/stars/Turbo31150/jarvis-linux?style=flat-square&color=6366f1)](https://github.com/Turbo31150/jarvis-linux)

A 9-layer autonomous operating system spanning boot to voice, running **600+ AI agents** across a 6-GPU cluster. JARVIS coordinates multi-model inference, voice pipelines, trading engines, browser automation, and self-healing domino chains — all on Linux.

| Metric | Value |
|--------|-------|
| **Autonomous Agents** | 600+ MCP handlers across distributed nodes |
| **Voice Commands** | 2,658 recognized commands, Whisper CUDA (<300ms) |
| **MCP Tools** | 87 tools orchestrated via Claude Agent SDK |
| **Domino Chains** | 835 automation pipelines, self-healing |
| **Trading Engine** | 6-model consensus, MEXC Futures, 800+ pairs |
| **Inference** | 6 NVIDIA GPUs / 46GB VRAM, LM Studio + Ollama |

```
Architecture:
Layer 9  ─  Vocal / Conversational Interface
Layer 8  ─  Multi-Agent Orchestration
Layer 7  ─  Trading Consensus Engine
Layer 6  ─  Browser & Web Automation
Layer 5  ─  MCP Tool Registry
Layer 4  ─  GPU Cluster Management
Layer 3  ─  Domino Pipeline Engine
Layer 2  ─  systemd Service Layer
Layer 1  ─  Linux Boot Integration
```

**Related repos:**
[jarvis-cowork](https://github.com/Turbo31150/jarvis-cowork) — 570+ autonomous QA scripts, continuous self-repair |
[Linux-Jarvis-Podcats](https://github.com/Turbo31150/Linux-Jarvis-Podcats) — AI podcast generation from cluster logs

### TradeOracle — Multi-Model Consensus Trading

[![Repo](https://img.shields.io/badge/GitHub-TradeOracle-181717?style=flat-square&logo=github)](https://github.com/Turbo31150/TradeOracle)
[![Python](https://img.shields.io/badge/Python-3776AB?style=flat-square&logo=python&logoColor=white)](https://github.com/Turbo31150/TradeOracle)
[![Stars](https://img.shields.io/github/stars/Turbo31150/TradeOracle?style=flat-square&color=6366f1)](https://github.com/Turbo31150/TradeOracle)

Algorithmic trading engine using **6-model AI consensus** (Claude, Gemini, GPT, DeepSeek, Mistral, Qwen) for signal generation. Detects BREAKOUT, REVERSAL, and MOMENTUM patterns with automatic TP/SL placement on MEXC Futures across 800+ trading pairs.

| Feature | Detail |
|---------|--------|
| **Consensus** | 6 AI models vote on every signal — majority required |
| **Patterns** | BREAKOUT / REVERSAL / MOMENTUM detection |
| **Exchange** | MEXC Futures, real-time execution |
| **Alerts** | Telegram notifications, dashboard integration |
| **Backtesting** | Monte Carlo simulations via Nexus-Elastic |

**Related repos:**
